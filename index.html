A retenir

Une adresse IP est constituée de deux parties : l'adresse du réseau et l'adresse de la machine qui permet donc de distinguer une machine sur un réseau.

Deux machines se trouvant sur un même réseau possèdent la même adresse réseau mais pas la même adresse machine.

Dans notre cas :

Le masque sous réseau 255.255.255.0 indique que les trois premiers nombres sont utilisés pour l'adresse réseau et le dernier nombre pour l'adresse machine.

Les machines M1, M2 et M3 sont dans le réseau dont l'adresse est : 192.168.1.0

M4 est dans le réseau dont l'adresse est : 192.168.2.0
  1. Construire 2 réseaux de 3 machines.

Le réseau des machines M1, M2, M3 a pour adresse 192.168.1.0.

Le réseau des machines M4, M5, M6 a pour adresse 192.168.2.0.

Pour relier les 2 réseaux, il faut ajouter un routeur (avec au moins 2 interfaces réseaux puisqu'il y a 2 réseaux).

Définition : Un routeur est un équipement qui permet de relier différents réseaux entres eux

  1. Rendu TP : Faire une capture d'écran de votre réseau et la coller dans le document de rendu

Il faut maintenant paramétrer les deux interfaces du routeur. La première interface doit correspondre au premier réseau et l'autre au deuxième réseau.

Par convention, on a tendance à affecter au routeur une adresse en fin de plage IP. Par exemple, sur le réseau 192.168.1.0/24, on gardera souvent l'adresse IP 192.168.1.254 pour le routeur.

Les passerelles

Il faut maintenant ajouter l'adresse IP du routeur dans le champ Passerelle (Gateway en anglais) de chaque machine de chaque réseau.

Quand un ordinateur tente de communiquer avec un autre périphérique via le protocole TCP/IP, il détermine si la machine de destination appartient au même réseau ou pas.

  • Si la machine est dans le réseau local alors l'ordinateur envoie le paquet sur le réseau local en transitant par le switch
  • Si la machine n'est pas dans le réseau local alors l'ordinateur transmet le paquet à la passerelle (le routeur) définie dans ses propriétés TCP/IP.
  1. Cliquer sur chaque machine et paramétrer le champ "passerelle"

  1. Une fois que tout est bien complété, passez en mode simulation.
  2. Installer une console sur M1, M3 et M5.
  3. Depuis M1, faites un ping sur M3, puis sur M5.
  4. Rendu TP : Faire une capture d'écran de votre ligne de commande
  5. Depuis M5 faites un ping sur M4 puis sur M1
  6. Rendu TP : Faire une capture d'écran de votre ligne de commande

Si tout a bien été configuré, vous devez constater que les machines des 2 réseaux peuvent communiquer entre-elles aussi bien dans le réseau local qu'entre les 2 réseaux.
